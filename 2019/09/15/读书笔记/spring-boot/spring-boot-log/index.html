Spring Boot SLF4J Log级别

SLF4J Log级别分为以下几种

  • debug：调试，一般作为最低级别
  • info： 输出重要的信息、提示信息
  • warn： 警告
  • error：错误信息

问题描述

在开发过程中，为了方便调试或某种原因，将过程数据直接打印输出，在服务实际运行中，中间过程数据过多，导致日志输出很多，同时服务占用内存急剧上升

解决办法

  • 控制好日志级别，有些日志仅在开发过程中查看，可以降低级别，上线时配置日志打印级别忽略此类型日志
  • 避免在程序中过多打印中间数据
